# 工具箱升级说明

## 开源 Code Agent 调研结论

成熟 code agent 的稳定工具层通常包括：文件读写、内容检索、文件匹配、受限 shell、任务/待办、子任务、浏览器或网页抓取、代码库地图、编辑/补丁、测试运行、权限/安全拦截。内网垂类智能体不应默认启用网页抓取和外部 MCP，应优先建设离线可控的文件、代码、文档、笔记、图谱、版本、本地检查和领域工具。

## 新工具箱

- 文件工具箱 files：工作区文件 IO、差异、补丁、移动、删除。
- 代码检索工具箱 code：glob、grep、窗口读取、符号抽取、repo_map。
- 文档读取工具箱 docs：离线文本抽取、元数据、表格预览。
- 笔记工具箱 notes：note.md 检索、读取、检查、创建、更新。
- 图谱索引工具箱 graph：图谱百科检索、邻域读取、全量重建、健康检查。
- 版本治理工具箱 version：状态、提交、历史、差异、恢复、发布、回退。
- 本地检查工具箱 shell：受限命令安全检查和运行。
- 领域工具箱 nx / design_report：保留垂类 CAD 与报告能力。

## 删除的旧工具

删除 cloude、textops、workspace、system/*、workflow/* 等与新基础工具职责冲突或当前无效的旧工具。后续如需恢复，应以新工具箱边界重新建模。
